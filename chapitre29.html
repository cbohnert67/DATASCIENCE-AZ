Chapitre 29 : Introduction aux APIs - Collecter des données en temps réel sur le web

Toutes les données ne sont pas sagement rangées dans des fichiers CSV statiques. Sur le web, les données sont vivantes : le cours d'une action change chaque seconde, la météo est mise à jour en permanence, de nouveaux tweets sont publiés constamment. Les APIs (Interfaces de Programmation d'Application) sont des ponts qui nous permettent de nous brancher directement sur ces services web pour obtenir des données fraîches en temps réel. Maîtriser les APIs, c'est passer du statut de consommateur de données passif à celui de collecteur actif.

Dialoguer avec le web

Qu'est-ce qu'une API REST ?

L'analogie du restaurant est parfaite pour comprendre. Une API est un intermédiaire structuré qui permet à deux applications de se parler sans avoir à connaître les détails complexes de l'autre.

  • Vous (le client) = Votre script Python.
  • Le serveur (le waiter) = L'API elle-même, qui reçoit vos requêtes.
  • La cuisine (le backend) = Le serveur distant où les données sont stockées et traitées.
  • Le menu = La documentation de l'API, qui vous dit quelles "commandes" (requêtes) vous pouvez passer.
[Image d'un diagramme du cycle requête-réponse d'une API]

Une API REST est simplement une manière populaire et standardisée de construire ces "menus" et d'organiser ces "commandes" en utilisant le protocole HTTP standard du web, ce qui les rend universelles et faciles à utiliser.

L'anatomie d'une requête HTTP

Chaque "commande" que vous passez à l'API est une requête HTTP. Prenons un exemple concret avec l'API météo : "Je veux la température horaire pour Paris".

  • La méthode : Le verbe d'action. Ici, on veut obtenir des informations, donc on utilise `GET`. Si on voulait publier un tweet, on utiliserait `POST`.
  • L'URL (endpoint) : C'est l'adresse de la "cuisine" spécialisée dans les prévisions. Par exemple : `https://api.open-meteo.com/v1/forecast`.
  • Les paramètres : Ils précisent votre commande. On les ajoute après un `?` et on les sépare par des `&`.
    • Pour Paris : `latitude=48.85&longitude=2.35`
    • Pour la température horaire : `hourly=temperature_2m`

    URL complète : `https://api.open-meteo.com/v1/forecast?latitude=48.85&longitude=2.35&hourly=temperature_2m`

  • Les en-têtes (headers) : Ce sont des instructions pour le serveur. La plus courante est la clé d'API pour l'authentification.

    `Authorization: Bearer VOTRE_CLÉ_SECRÈTE`

    (Heureusement, l'API météo de notre exemple n'en a pas besoin !)

La réponse du serveur

Une fois la requête envoyée, le serveur vous renvoie votre "plat".

1. Le code de statut

C'est la première réponse du serveur : "J'ai bien reçu, voici le statut".

  • 200 OK: "Voici votre plat, tout est parfait."
  • 404 Not Found: "Le plat que vous demandez n'est pas au menu."
  • 403 Forbidden: "Vous n'êtes pas autorisé à commander dans cette section du restaurant."

2. Le corps (body) en JSON

Si tout va bien (code 200), voici les données, souvent au format JSON. C'est un format texte qui est presque identique à un dictionnaire Python.

{
  "latitude": 48.85,
  "hourly": {
    "time": ["2023-10-26T00:00", "..."],
    "temperature_2m": [12.5, 12.3, "..."]
  }
}

On voit bien la structure : une clé `"hourly"` qui contient un autre dictionnaire, avec des clés (`"time"`, `"temperature_2m"`) dont les valeurs sont des listes.

Interroger une API Météo avec Python

La bibliothèque `requests` est le moyen le plus simple et le plus populaire pour faire des requêtes HTTP. Voyons étape par étape comment l'utiliser pour obtenir les prévisions météo pour Paris.

Étape 1 : Définir la "commande"

On prépare notre requête. On sépare l'URL de base des paramètres. C'est une bonne pratique qui rend le code plus lisible. La bibliothèque `requests` assemblera l'URL finale pour nous.

import requests
import pandas as pd
import matplotlib.pyplot as plt

base_url = "https://api.open-meteo.com/v1/forecast"
params = {
    "latitude": 48.85,
    "longitude": 2.35,
    "hourly": "temperature_2m"
}

Étape 2 : Envoyer la requête et gérer les erreurs

On envoie la requête avec `requests.get()`. Le bloc `try...except` est une sécurité essentielle : si le site est en panne ou si notre connexion internet est coupée, le programme affichera un message d'erreur propre au lieu de planter. `response.raise_for_status()` est un garde-fou : il vérifiera si le code de statut est un code d'erreur (comme 404 ou 500) et, si c'est le cas, arrêtera le script en signalant l'erreur.

try:
    response = requests.get(base_url, params=params)
    response.raise_for_status()
    # ... la suite du code si tout va bien ...
except requests.exceptions.RequestException as e:
    print(f"Erreur lors de la requête : {e}")

Étape 3 & 4 : Extraire et Structurer les données

La réponse brute est du texte. `response.json()` la convertit en un dictionnaire Python. En inspectant ce dictionnaire, on voit que les données qui nous intéressent sont dans la clé `'hourly'`. On peut alors passer ce sous-dictionnaire directement à `pd.DataFrame()` pour le transformer en un tableau structuré. On en profite pour convertir la colonne 'time' en un vrai format de date, ce qui est crucial pour les graphiques et les analyses temporelles.

    data = response.json()
    df_weather = pd.DataFrame(data['hourly'])
    df_weather['time'] = pd.to_datetime(df_weather['time'])
    print(df_weather.head())

Étape 5 : Analyser et Visualiser

Maintenant que les données sont dans un DataFrame Pandas, le plus dur est fait ! On peut utiliser toutes les connaissances que nous avons acquises pour tracer des graphiques avec Matplotlib et analyser les tendances.

Challenge pour vous !

Les APIs permettent de faire des choses très amusantes. Utilisons "The Dog API" pour égayer notre journée.

Votre mission :

Utilisez la bibliothèque `requests` pour interroger "The Dog API" afin de récupérer une image aléatoire d'une race de chien spécifique (par exemple, un Beagle). Ensuite, affichez cette image.

Cliquez ici pour la solution 
import requests
from IPython.display import Image, display

# 1. Trouver l'ID de la race "Beagle"
breeds_url = "https://api.thedogapi.com/v1/breeds/search?q=beagle"
breeds_response = requests.get(breeds_url)
beagle_id = breeds_response.json()[0]['id']

# 2. Demander une image pour cet ID
image_url = f"https://api.thedogapi.com/v1/images/search?breed_ids={beagle_id}"
image_response = requests.get(image_url)

if image_response.status_code == 200:
    random_beagle_url = image_response.json()[0]['url']
    print(f"URL de l'image : {random_beagle_url}")
    # 3. Afficher l'image
    display(Image(url=random_beagle_url))
else:
    print("Erreur !")
