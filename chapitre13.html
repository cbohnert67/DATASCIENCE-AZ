Chapitre 13. Machine Learning - Mon ordinateur peut-il vraiment apprendre ?

Comment apprend-on à un très jeune enfant à reconnaître un chat ? On ne lui donne pas une liste de règles. À la place, on lui montre des exemples. On lui montre une photo et on lui dit "ça, c'est un chat". On lui montre un chien : "ça, ce n'est pas un chat". À force d'exemples, l'enfant finit par construire son propre modèle mental et par généraliser. Le Machine Learning (ou Apprentissage Automatique), c'est exactement la même idée : au lieu de programmer des instructions explicites, nous nourrissons un algorithme avec des données, et nous le laissons "apprendre" les règles par lui-même.

Le vocabulaire indispensable de l'apprenti sorcier

Features (ou variables)

Ce sont les "indices" ou les "caractéristiques" que nous donnons au modèle pour qu'il puisse prendre une décision. Ce sont les colonnes de notre jeu de données que nous utilisons comme entrées. Mathématiquement, on les représente souvent par un vecteur \(X\).

Exemple Titanic : Pour un passager, les features pourraient être sa classe (Pclass), son sexe (Sex) et son âge (Age).

# 'X' est un DataFrame contenant nos features
features = ['Pclass', 'Sex', 'Age']
X = df[features]
print(X.head())

Label (ou cible)

C'est la "bonne réponse" que nous voulons que le modèle apprenne à prédire. C'est la colonne cible de notre analyse. Mathématiquement, on la représente souvent par un vecteur \(y\).

Exemple Titanic : Le label est la colonne 'Survived', qui indique si le passager a survécu (1) ou non (0).

# 'y' est une Series contenant notre label
y = df['Survived']
print(y.head())

Modèle

C'est le "cerveau" que l'algorithme construit après avoir analysé les données. C'est une fonction mathématique \(f\) qui a appris à approximer la relation entre les features \(X\) et le label \(y\).

\[ y \approx f(X) \]

Exemple : Dans une régression linéaire simple, le modèle est juste l'équation d'une droite \(y = ax + b\). L'algorithme apprend les meilleures valeurs pour \(a\) (la pente) et \(b\) (l'ordonnée à l'origine).

Entraînement (Training)

C'est le processus durant lequel le modèle ajuste ses paramètres internes pour minimiser son erreur. C'est la phase où l'on montre les exemples (\(X\)) et les réponses (\(y\)) à l'algorithme. Il compare ses prédictions \(\hat{y} = f(X)\) aux vraies réponses \(y\) et ajuste \(f\) pour réduire l'écart, souvent via la descente de gradient.

Exemple avec Scikit-learn : La méthode .fit() est la fonction qui lance tout le processus d'entraînement.

from sklearn.linear_model import LogisticRegression

# 1. Créer une instance du modèle
model = LogisticRegression()

# 2. Entraîner le modèle en lui montrant les features (X) et le label (y)
model.fit(X_train, y_train)

Les 3 grandes familles de l'apprentissage

1. L'Apprentissage Supervisé (Apprendre avec un professeur)

C'est la méthode la plus courante. On entraîne un modèle sur des données où la "bonne réponse" (le label) est fournie. Le but est de généraliser pour pouvoir prédire le label de nouvelles données, jamais vues auparavant.

La Classification

Le but est de prédire une catégorie discrète. La réponse est "quoi ?" ou "est-ce que ?". Exemples : Ce mail est-il un spam ou non ? Quelle est l'espèce de cette fleur ?

Notre projet Titanic : Prédire si un passager a survécu (catégorie 1) ou non (catégorie 0).

La Régression

Le but est de prédire une valeur numérique continue. La réponse est "combien ?". Exemples : Quel sera le prix de cet appartement ? Quelle sera la température demain ?

2. L'Apprentissage Non Supervisé (Apprendre par soi-même)

Ici, on ne donne pas les réponses (pas de labels). On donne au modèle des données brutes et on lui demande de trouver une structure cachée, de mettre de l'ordre dans le chaos. Le cas le plus courant est le clustering.

Exemple : On donne à un modèle les données d'achat de milliers de clients. Le modèle va créer des groupes (clusters) de clients aux comportements similaires (les "acheteurs fréquents", les "chasseurs de promos") sans qu'on lui ait défini ces groupes au préalable.

from sklearn.cluster import KMeans
# X sont les données non étiquetées
kmeans = KMeans(n_clusters=3)
kmeans.fit(X) # Notez qu'il n'y a pas de 'y' ici !
clusters = kmeans.predict(X)

3. L'Apprentissage par Renforcement (Apprendre par l'expérience)

C'est le mode d'apprentissage le plus proche de la manière dont nous, humains, apprenons. L'algorithme ("l'agent") apprend par essais et erreurs. Il évolue dans un environnement, prend des actions, et reçoit en retour une récompense (positive) ou une punition (négative). Son seul but est de trouver une stratégie (une "politique") qui maximise la somme des récompenses sur le long terme.

Exemple : Entraîner une IA à jouer à un jeu vidéo. L'IA essaie des actions au hasard. Si elle gagne des points (récompense), elle essaiera de refaire cette action dans une situation similaire. Si elle perd une vie (punition), elle évitera cette action à l'avenir.
